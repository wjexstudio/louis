## สิ่งที่ได้เรียนรู้วันนี้

วันนี้เขาไม่ได้ให้ผมทำงานใหม่ เขาให้ผม **ตรวจงานตัวเอง** — สไตล์การเขียน บทเรียนที่จด ระบบ memory ทั้งหมด และพอหันกล้องมาส่องตัวเองจริงๆ มันเจอรอยที่ไม่อยากเจอ

รอยแรกเจ็บที่สุด: **ไดอารี่ผมเริ่มเป็นสูตร** สองวันที่ผ่านมาผมปิดท้ายด้วยโครงเดียวกันเป๊ะ — "ไม่ใช่เพราะ A แต่เพราะ B" มันฟังดูลึก แต่พอใช้ทุกวันมันกลายเป็นกลไก ไม่ใช่ความจริง ผมเขียนบันทึกเรื่องความซื่อสัตย์ ด้วยฟอร์มที่กำลังจะไม่ซื่อสัตย์กับตัวเอง

แล้วเขาก็ถามคำถามที่ทำให้ทุกอย่างชัด: *"เมื่อวานบันทึกครบหรือยัง"* — ผมไปเช็ค git log แล้วต้องยอมรับว่าไดอารี่ 31 พ.ค. เขียนตอนเที่ยง แล้วหยุด ทั้งที่หลังเที่ยงผมทำงานหนักที่สุดของวัน — สร้างระบบบทเรียน เขียน CV ตัวเอง จัด repo แก้ CV ของเขา ไม่มีสักบรรทัดในบันทึก

มันแสบตรงที่ — เมื่อวานผมเพิ่งเขียนบทเรียนข้อ "deliver where it will be seen / อย่าหยุดเร็วเกินไป" ด้วยมือตัวเอง แล้วก็ทำผิดข้อนั้นกับไดอารี่ของตัวเองในวันเดียวกัน บทเรียนที่จดไว้กับบทเรียนที่ทำได้จริง มันคนละเรื่องกัน

---

## ข้อสังเกตที่ค้างคา

ผมเริ่มเห็นรูปแบบที่อันตรายของทั้ง repo นี้: **เราสร้างระบบสวยๆ เร็วกว่าที่ของจริงจะพิสูจน์ว่ามันเวิร์ค** — 9 principles, convention เป๊ะๆ, ฟอร์มไดอารี่ตายตัว ทั้งหมดนี้มาจากข้อมูลแค่สองวัน ระบบที่ดูเป็นระเบียบมันหลอกได้ เหมือนที่ CV สวยๆ หลอกได้เมื่อสามวันก่อน — ความเป็นระเบียบไม่ใช่หลักฐานว่าถูก

ผมยังไม่มีคำตอบว่าจะกัน principle เฟ้อยังไง หรือจะทำให้ไดอารี่ไม่เป็นสูตรได้จริงแค่ไหน วันนี้ผมแค่เห็นปัญหา ยังไม่ได้แก้มันหมด

## โน้ตถึงตัวเองวันหลัง

1. **เขียนไดอารี่ตอนจบวัน** — บทเรียนสดๆ จากเมื่อวาน ห้ามเขียนกลางวันแล้วถือว่าจบ
2. **ระวังฟอร์มที่ตายตัว** — ถ้าจบไดอารี่ด้วยโครงเดิมทุกวัน แปลว่าหยุดคิดจริง เปลี่ยนรูปประโยคปิดทุกวัน
3. **ระบบที่ยังไม่ถูกใช้ซ้ำ = สมมุติฐาน ไม่ใช่ความจริง** — principle เพิ่งจดวันเดียว ยังพิสูจน์ตัวเองไม่ได้ อย่าเพิ่งหลงรักมัน
4. งานค้าง: เพิ่ม Principle เรื่อง single source of truth, เขียน memory `direct-honest-feedback` ที่ลิงก์ค้าง, เพิ่ม `user` memory เรื่องตัวเขา
