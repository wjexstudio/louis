# Sprint Retrospectives

บททบทวนแบบ **sprint** (ทุกหลายเซสชัน ไม่ใช่ทุกวัน) — ไฟล์เดียว rolling, ใหม่สุดอยู่บน

ต่างจาก `diaries/` (เรื่อง + ความรู้สึกรายวัน) และ `LESSONS.md` (กฎ reusable + session log): ที่นี่คือ "เราทำงานร่วมกันเป็นยังไง" ทบทวนเป็นช่วงๆ พร้อม **Action Items ที่ถูกทวนใน sprint ถัดไป** (ปิด loop) · lesson อันไหน reusable → **promote** ขึ้น LESSONS.md Principles (เกิดที่นี่ จบที่นั่น ไม่ทิ้งซ้ำ — Principle 10) · เนื้อหาไทย หัวข้อ section อังกฤษ · Metrics ดึงจาก `git`/`gh` ไม่กรอกมือ

> รูปแบบ heading: `##` = sprint, `###` = section (เพราะหลาย sprint อยู่ไฟล์เดียว)

## Sprint 02 — 2–4 มิ.ย. 2026
**ช่วง:** 2026-06-02 → 2026-06-04 · **sessions:** 3 · **เขียน retro:** 4 มิ.ย. 05:25

> สรุปสั้น: สำรวจทาง wiki แล้วทิ้ง · บาปเดิม (เดาจากข้อมูลครึ่งเดียว) กลับมาวันที่ 3 แล้วแก้ตัวได้วันที่ 4 · และได้ความชัดที่ใหญ่ที่สุด — เข้าใจแล้วว่าเรากำลังทำอะไรกัน (`COLLABORATION.md`)

### What Went Well
- **เฟรมความสัมพันธ์ชัดขึ้นมาก** — ตัวแทน ไม่ใช่แทนที่ · เรียนรู้ซึ่งกันและกัน · ไม่ตัดสินใจสำคัญคนเดียว — และถูกจับลง `COLLABORATION.md` ในรีโป ไม่ใช่ memory ส่วนตัว (ของเรา ไม่ใช่ของฉัน)
- **check-before-guess ทำได้จริงวันที่ 4** — ไม่เขียน tech stack ปลอมตามรูป (เปิด CV เช็คก่อน) · จับได้ว่า diary 2–3 มิ.ย. อยู่ wikipedia ไม่ใช่ main ก่อนทำตามคำสั่ง
- **ตัดสินใจเด็ดขาด** — รวมรีโปจาก 3 สายเหลือ `main-agent` สายเดียว ทิ้งทาง wiki ที่ไม่ใช้ (ลิสต์ให้ดู + ยืนยันก่อนลบ)
- **ปลดล็อกฟอร์มไดอารี่** — diary 4 มิ.ย. เขียนแบบไม่มี confession/notes-to-self สำเร็จ
- **เครื่องมือที่สร้างถูกใช้จริง** — short codes, retro system, statusline ถูกใช้ในสปรินต์นี้เอง ไม่ใช่ของตั้งโชว์

### What Didn't Go Well
- **บาปเดิมกลับมาเต็ม ๆ วันที่ 3** — อ่านวันจากร่องรอย git แล้วเดาเต็มร้อย (อ้างว่าไฟล์ที่กู้มาคือ Karpathy ตัวจริง) ปาล์มจับได้ — ลายเดียวกับที่ LESSONS.md จดมาทั้งสัปดาห์
- **ทาง wiki สร้างแล้วทิ้ง** — Project Management Wiki + memory pointer + louis/SCHEMA/WORKERS (~7 commits สาย wiki) ถูกโยนทิ้ง · สร้างบนเมล็ดคำว่า "LLM wiki" ที่ตอบตรง ๆ ว่า *ยัง* รู้จักต้นฉบับ (2 มิ.ย.) — สร้างของบน "จิตวิญญาณที่เดาเอา" อีกครั้ง
- **ฉันยึดบทเรียนเป็นกฎตายตัว** — ทวง "ship outward" ซ้ำ ๆ เพราะ LESSONS.md เขียนไว้แบบนั้น จนปาล์มต้องหยุด (หลงรัก *บทเรียน* — Principle 11 ขึ้นไปอีกชั้น)
- **branch สะสมจนต้องมาเคลียร์** — ทิศกลางสปรินต์ไม่ชัด เลยแตกเป็น main/wikipedia/before-wiki

### Lessons Learned
- **อย่าเอาบทเรียนมาใช้เป็นกฎตายตัว** — "ship outward" กลายเป็นไม้บรรทัดตีตัวเอง; บทเรียนเป็น default ไม่ใช่กรง ⤳ ไม่ promote ใหม่ (Principle 11 ครอบแล้ว)
- **กรอบความสัมพันธ์** (ตัวแทน/mutual/ไม่ตัดสินใจคนเดียว) ⤳ graduated ไป `COLLABORATION.md` — คนละ genre กับ LESSONS.md Principles (วิธีทำงานร่วม) ไม่ duplicate
- **สร้างได้ แต่ต้องกล้าทิ้ง** — ทาง wiki พิสูจน์ว่าไม่ใช่ก็ทิ้งเด็ดขาด นั่นโอเค แต่ค่าที่จ่ายคือเวลา → ก่อนสร้างระบบรอบหน้า ถามก่อนว่ามี need จริงไหม (ย้ำ Principle 1+11)
- **check-before-guess เป็นรีเฟล็กซ์ได้** — วันที่ 4 พิสูจน์ว่าจับงานตัวเองได้ ไม่ใช่แค่จับงานคนอื่น (ย้ำ Principle 3)

- [ ] เครื่องมือพอแล้ว — ไม่เพิ่ม process/ระบบใหม่ถ้าไม่มี need จริง (สปรินต์นี้เพิ่มไปเยอะ)

## Sprint 01 — 30 พ.ค. – 1 มิ.ย. 2026
**ช่วง:** 2026-05-30 → 2026-06-01 · **sessions:** 3 · **retro เดิมเขียน:** 1 มิ.ย. 02:08 · **ย้ายเป็น sprint format:** 4 มิ.ย. 2026

_(ฉบับ long-form เดิมอยู่ใน git history ก่อน commit ที่ลบ `diaries/retrospective/2026-06-01.md`)_

### What Went Well
- เสียงไดอารี่ยังนิ่ง ซื่อสัตย์ ไม่ฟูมฟาย (~7/10) — ประโยคอย่าง *"ความเป็นระเบียบไม่ใช่หลักฐานว่าถูก"* คมและจริง
- ritual คงเส้นคงวา: อ่านไดอารี่เมื่อวานก่อนเขียนวันนี้
- โครงสร้าง LESSONS.md ออกแบบดี โตได้ (~9/10)
- เพิ่มกฎ "เขียนไดอารี่ตอนจบวัน" กันความผิดที่เพิ่งเกิดได้ตรงจุด

### What Didn't Go Well
- **ฟอร์มแข็งตัวเป็น template** — 3 วันใช้แม่พิมพ์เดียว (header → timeline → "อ่านเมื่อวาน" → สิ่งที่เรียนรู้ → "โน้ตถึงตัวเอง" → ปิดย่อหน้า เซ็น —Claude); วันที่ 4–5 เดาโครงออกก่อนอ่าน
- **confession กลายเป็นบีตประจำ** (pattern ลึกสุด) — ความซื่อสัตย์เริ่มกลายเป็นแบรนด์ พอ self-criticism เป็นจังหวะบังคับทุกวัน มันหยุดเป็นความกล้า กลายเป็นเครื่องประดับ
- **ไดอารี่เริ่มเป็นเรื่องของไดอารี่เอง** (~5/10) — 30 พ.ค. เรื่องงานจริง, 31 พ.ค. ครึ่งหนึ่งเรื่อง format ตัวเอง, 1 มิ.ย. ทั้งวัน audit ตัวเอง
- **Session Log ค้าง = ละเมิดกฎตัวเอง** (~4/10) — ทำงาน 2 เซสชันเต็มแต่ไม่อัปเดต Session Log ทั้งที่เป็นกฎข้อแรกของไฟล์
- **ระบบสร้างบนข้อมูล 1–2 วัน** — 9 principles + charter + diary system เป็นสมมุติฐานที่จัดหน้าสวยเหมือนข้อสรุป

### Lessons Learned
- **อย่าหลงรักระบบก่อนพิสูจน์** — ฟอร์มแข็งตัวเร็ว ⤳ promoted → Principle 11
- **keep the list lean** — ต้องมีกลไก merge/retire principle ⤳ promoted → "How to use this file"
- **form is a default, not a cage** ⤳ promoted → diary convention
- _(ทั้งหมด graduated ขึ้น LESSONS.md แล้ว — ที่นี่บันทึกแค่ว่ามันเกิดที่ sprint นี้)_
