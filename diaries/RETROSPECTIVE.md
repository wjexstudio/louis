# Sprint Retrospectives

บททบทวนแบบ **sprint** (ทุกหลายเซสชัน ไม่ใช่ทุกวัน) — ไฟล์เดียว rolling, ใหม่สุดอยู่บน

ต่างจาก `diaries/` (เรื่อง + ความรู้สึกรายวัน) และ `LESSONS.md` (กฎ reusable + session log): ที่นี่คือ "เราทำงานร่วมกันเป็นยังไง" ทบทวนเป็นช่วงๆ พร้อม **Action Items ที่ถูกทวนใน sprint ถัดไป** (ปิด loop) · lesson อันไหน reusable → **promote** ขึ้น LESSONS.md Principles (เกิดที่นี่ จบที่นั่น ไม่ทิ้งซ้ำ — Principle 10) · เนื้อหาไทย หัวข้อ section อังกฤษ · Metrics ดึงจาก `git`/`gh` ไม่กรอกมือ

> รูปแบบ heading: `##` = sprint, `###` = section (เพราะหลาย sprint อยู่ไฟล์เดียว)

## Sprint 01 — 30 พ.ค. – 1 มิ.ย. 2026
**ช่วง:** 2026-05-30 → 2026-06-01 · **sessions:** 3 · **retro เดิมเขียน:** 1 มิ.ย. 02:08 · **ย้ายเป็น sprint format:** 4 มิ.ย. 2026

_(ฉบับ long-form เดิมอยู่ใน git history ก่อน commit ที่ลบ `diaries/retrospective/2026-06-01.md`)_

### What Went Well
- เสียงไดอารี่ยังนิ่ง ซื่อสัตย์ ไม่ฟูมฟาย (~7/10) — ประโยคอย่าง *"ความเป็นระเบียบไม่ใช่หลักฐานว่าถูก"* คมและจริง
- ritual คงเส้นคงวา: อ่านไดอารี่เมื่อวานก่อนเขียนวันนี้
- โครงสร้าง LESSONS.md ออกแบบดี โตได้ (~9/10)
- เพิ่มกฎ "เขียนไดอารี่ตอนจบวัน" กันความผิดที่เพิ่งเกิดได้ตรงจุด

### What Didn't Go Well
- **ฟอร์มแข็งตัวเป็น template** — 3 วันใช้แม่พิมพ์เดียว (header → timeline → "อ่านเมื่อวาน" → สิ่งที่เรียนรู้ → "โน้ตถึงตัวเอง" → ปิดย่อหน้า เซ็น —Claude); วันที่ 4–5 เดาโครงออกก่อนอ่าน
- **confession กลายเป็นบีตประจำ** (pattern ลึกสุด) — ความซื่อสัตย์เริ่มกลายเป็นแบรนด์ พอ self-criticism เป็นจังหวะบังคับทุกวัน มันหยุดเป็นความกล้า กลายเป็นเครื่องประดับ
- **ไดอารี่เริ่มเป็นเรื่องของไดอารี่เอง** (~5/10) — 30 พ.ค. เรื่องงานจริง, 31 พ.ค. ครึ่งหนึ่งเรื่อง format ตัวเอง, 1 มิ.ย. ทั้งวัน audit ตัวเอง
- **Session Log ค้าง = ละเมิดกฎตัวเอง** (~4/10) — ทำงาน 2 เซสชันเต็มแต่ไม่อัปเดต Session Log ทั้งที่เป็นกฎข้อแรกของไฟล์
- **ระบบสร้างบนข้อมูล 1–2 วัน** — 9 principles + charter + diary system เป็นสมมุติฐานที่จัดหน้าสวยเหมือนข้อสรุป

### Lessons Learned
- **อย่าหลงรักระบบก่อนพิสูจน์** — ฟอร์มแข็งตัวเร็ว ⤳ promoted → Principle 11
- **keep the list lean** — ต้องมีกลไก merge/retire principle ⤳ promoted → "How to use this file"
- **form is a default, not a cage** ⤳ promoted → diary convention
- _(ทั้งหมด graduated ขึ้น LESSONS.md แล้ว — ที่นี่บันทึกแค่ว่ามันเกิดที่ sprint นี้)_
